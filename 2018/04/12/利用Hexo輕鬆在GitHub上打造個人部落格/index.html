為什麼選用Hexo?

靜態網站產生器有 Jekyll、 Hexo，為什麼我選用Hexo呢？原因很簡單，因為我覺得Hexo提供的Theme比較好看(超任性)，至於還有像是WordPress之類的，考量還要租用空間，我只是想單純的當作筆記，順便分享出來我辛苦看英文的成果，廢話不多說，我們來開始吧

在GitHub上開啟專案

首先需要先擁有[GitHub]的帳號，註冊完成後可以看到下圖，點擊Start a project

為自己的Project取個名稱，之後如果沒有購買自己的網域話，這個會變成你的網址


看到這畫面代表開專案完成囉，先別關閉畫面，等等需要用到

安裝

Node.js

Node.js介紹可以看這裡，為什麼要安裝Node.js呢？因為Hexo是由Node.js編寫而成。
首先Node.js官網下載安裝，一路下一步即可安裝完成。

接下來要檢查是否安裝，打開終端機輸入

挑選喜歡的Theme

Hexo官網提供了200種主題任君挑選，找一個自己喜歡的主題，下載到剛剛新增的資料夾

更換Theme

打開資料夾，在根目錄下找到_config.yml檔案，找到theme，換成剛剛下載的名字

上傳檔案到GitHub

安裝Git

安裝完成後修改_config.yml檔案，再次找到deploy，輸入剛剛在GitHub創好的網址。

在輸入以下指令，即可將檔案傳到GitHub上

終端機可能會跳出詢問你的帳號密碼，輸入即可開始上傳

部署部落格

在GitHub網頁上我們可以看到檔案都被傳上來了，在專案裡面選擇Settings

下拉找到GitHub ，選則master branch，並點選Save

再次往下拉會發現出現了網址，大功告成！！撒花啦，專屬部落格完成。如果你有自己的網域話，可以在Custom domin輸入自己的網域，網址開頭就不會有github.io這樣醜醜的網址拉。
